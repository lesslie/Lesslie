HEXO 搭建github pages 实践

2016-04-25
workhexo
github-pages, hexo, markdown

一、写在前面

作为开篇，总结下使用hexo搭建github pages的过程，并将搭建过程中遇到的问题及解决方法。闲言少叙，以后在记录过程中将会一直保持这种直入主题，一语中的的风格，批郤导窾、切中肯綮。github pages搭建框架很多，Jekyll、octpress基于ruby，hugo基于go语言，而hexo使用的node.js，选hexo，是因为之前接触过node.js，对其有所了解。

二、准备

在申请的github账号下，创建名为xxx.github.io的repository。具体步骤可通过度娘搜索，此类文章很多，如创建GitHub技术博客全攻略等。

三、环境搭建

依据hexo官网说明，依次安装git客户端和nodejs开发环境，链接如下：

  • git client
  • Node.js

1、git client 安装

git client安装就是一路默认，安装完成之后。配置用于用户校验的ssh_key。具体步骤如下

①生成rsa

$ ssh-keygen -t rsa -C "youname@example.com"
Generating public/private rsa key pair.
Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):
可以输入生成key的文件路径；一般直接回车即可，此时默认路径为当前用户文件夹下。
执行完成后
Enter passphrase (empty for no passphrase):<输入加密串>
Enter same passphrase again:<再次输入加密串>
输入加密串，即可在对应目录看到.ssh/id_rsa.pub。

②配置rsa

将id_rsa.pub用文本编辑器打开后，拷贝到xxx.github.io的repository的setting下deploy key里即可。
如图：

以上两个安装完成之后，继续安装hexo，hexo是基于Node.js，所有的yml（hexo使用的标记语言）都是运行在node.js环境下，只有在此环境下hexo才能根据yml生成所需的静态页面。

2、hexo 安装

$ npm install -g hexo-cli
nodejs安装完成后，此命令在windows cmd或者在git bash里执行hexo都能成功安装。至此，环境已搭建完毕。

四、hexo 创建blog

新建一个用于存放和创建blog的工作目录，然后右键——>git bash here。
输入下面命令初始化工作目录：
hexo init
继续输入命令，安装依赖包
npm install
至此，已完成空的hexo blog框架，目录如下：

其中node_modules存放的是node.js的依赖，scaffolds是将来新建文章可以存放的一些目录，source就是将来要编写md文件的目录所在，themes是存放使用主题的目录。
hexo可以通过配置文件使用theme文件下不同主题，这里使用的是从ppoffice的hexo-theme-icarus fork并做一些改动，地址在这里。

执行下面命令，就会将source目录下的md文件生成对应的静态html到public目录下。
hexo generate
启动服务器,通过http://localhost:4000/ 即可访问。需要特殊记录的是，通过localhost或本机回环IP 127.0.0.1可能访问不通，最后使用本机局域网IP才正常访问。
hexo server
接下来就是新建blog，可通过如下命令新建:
hexo new "your-blog-name"
执行后就可以看到名为”your-blog-name.md”的文件在source/_posts文件夹下生成。使用markdown语言编写该文件即可，如果对该语言不熟悉，推荐几款所见即所得的编辑器在线的web版mdeditor和脱机版软件小飞匠。
关于图片的引用有几种方式：一、使用插件hexo-asset-image，执行npm install hexo-asset-image进行安装，这种方式在新建文章的同时会新建该文章对应得图片文件夹，当generate时同样会将图片文件夹生成到public下对应的文章目录下。但是对于这种方式在使用中会是中文变成unicode，如下这样导致了搜索空间也是对Unicode做索引，不能搜索文本内容。二、将图片放在第三方的服务器上，这也是众多人推荐的方式，减少了github的压力。三、在source的文件夹下创建一个专门存放图片的文件夹如imgs，所有的图片都放在该文件夹，因为在generate后生成的public文件夹就是将来发布站点的根目录，所以在引用该文件时通过/imgs/xxx.png路径引用即可。
关于md中tags和categories，语法规则都是

在冒号后面有空格，当tags只有一个或categories只有一级时，可省略中括号。
当”your-blog-name.md”编写完成后，执行hexo generate命令即会将该文件在public下的post文件下生成对应的静态html。通过浏览器访问即可看到。
